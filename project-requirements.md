# Project Requirements

## Цель
Сделать Windows desktop-приложение для учета камер и устройств видеонаблюдения на объектах: склад, офис, магазин и другие.

## MVP
- CRUD объектов.
- CRUD этажей и зон.
- При создании нового объекта автоматически создаётся первая зона `Новая зона`, чтобы объект сразу был готов к работе на плане.
- Простой редактор плана.
- Устройства на плане:
  - `camera`
  - `nvr`
  - `dvr`
  - `switch`
  - `poe_switch`
- Карточка устройства с редактированием основных полей.
- Для камер: сектор обзора, RTSP, IP, логин, пароль, модель, серийный номер, место установки и ответственный.
- Пароль скрыт по умолчанию, но может быть временно показан пользователем.
- Копирование IP, RTSP и пароля.
- Кабельные связи между устройствами:
  - UTP
  - FTP
  - coaxial
  - power
- Polyline-маршрут кабеля с промежуточными точками и редактированием handles.
- Поиск по названию, IP, типу устройства, объекту, зоне, статусу и заметкам.
- Переключатель отображения IP на плане, чтобы быстро видеть IP-адреса устройств прямо на карте.
- Таблица всех устройств.
- Локальное сохранение данных в IndexedDB.
- Импорт/экспорт полного проекта в JSON.
- Экспорт устройств в CSV.
- Экспорт плана в JPG.

- При запуске приложение открывает последний сохранённый snapshot из IndexedDB и восстанавливает activeObjectId / activeFloorId, если они валидны.
- Если активный object или floor удалён, приложение выбирает первый доступный object и первый floor внутри него и сохраняет это в uiState.
- Если проектов и объектов ещё нет, приложение не показывает demo data и выводит пустой экран в центре canvas.
- Если проектов и объектов ещё нет, в центре canvas показывается CTA с кнопкой `+`, которая открывает файл проекта `JSON` с компьютера.
- Если проект уже не пустой, empty-state и центральная кнопка `+` не показываются.
- В меню `Файл` есть пункт `Новый проект`, который с подтверждением закрывает текущий проект и создаёт пустой snapshot; в диалоге доступны `Сохранить`, `Нет` и `Отмена`.
- Режим `Просмотр` по умолчанию и ручное включение режима `Редактирование`.

## Ограничения
- Не добавлять backend.
- Не подключать Supabase.
- Не делать пароль видимым по умолчанию.
- Не превращать редактор в CAD.
- Не переписывать Lovable UI с нуля.
- Не усложнять desktop MVP сетевой синхронизацией или учетными записями.

## UX requirements
- В режиме `Просмотр` пользователь может выбирать объекты, зоны, устройства и элементы, но не может случайно переместить, удалить или изменить данные.
- Редактирование должно быть явно включено.
- Правая панель должна открываться по selection и не сдвигать canvas.
- Пользователь может закрепить правую панель pin toggle.
- Клик по пустому canvas снимает selection и закрывает правую панель, если она не закреплена.
- Каждый этаж должен восстанавливать последнее оставленное пользователем положение и масштаб карты.
- Создание `Room` должно быть быстрым: один клик создает помещение 200x200 и возвращает tool в `select`.
- Кабель должен строиться понятным сценарием: single click добавляет точки, double click завершает маршрут.
