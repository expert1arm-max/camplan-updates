# QA Checklist

## Build
- [x] `npm run build`
- [x] `npm run lint`
- [x] `npm run dist:win`

## Electron startup
- [ ] Запустить свежий Windows installer `release\CCTV Manager Setup 0.1.1.exe`.
- [ ] Проверить, что UI открывается со стилями и без black screen.
- [ ] Проверить, что приложение стартует в режиме `Просмотр`.
- [ ] Переключиться в `Редактирование` и обратно.
- [ ] Открыть `Помощь -> О программе` и проверить, что версия совпадает с Electron runtime.
- [ ] Запустить `Помощь -> Обновить программу` и проверить, что приложение делает ручную проверку GitHub Releases.
- [ ] Проверить, что во время проверки обновления показывается loader.
- [ ] Проверить, что при найденном релизе виден progress bar и объём скачанного/общего размера, а после завершения скачивания показывается сообщение о готовом обновлении.
- [ ] Проверить, что если обновлений нет, выводится `Обновлено до последней версии`.

## Startup / hydration
- [ ] Проверить первый запуск без проекта: demo data не появляется, в центре canvas виден empty-state.
- [ ] Проверить, что в центре empty-state есть кнопка `+`, она открывает JSON проекта с компьютера и исчезает после появления объектов.
- [ ] Проверить, что центральная CTA-кнопка реально кликается и не блокируется overlay-слоем.
- [ ] Проверить, что кнопка `+` в левой панели открывает JSON-файл проекта с компьютера и импортирует его в текущее состояние.
- [ ] Проверить, что `Файл -> Новый проект` спрашивает подтверждение и предлагает `Сохранить`, `Нет` и `Отмена` перед очисткой текущего проекта.
- [ ] Проверить повторный запуск: открывается последний snapshot из IndexedDB.
- [ ] Проверить восстановление activeObjectId / activeFloorId при валидных сохранённых данных.
- [ ] Проверить fallback, если active object или floor удалён: выбирается первый доступный object и первый floor внутри него.

- [ ] Открыть карточку устройства.

- [ ] Показать и скрыть пароль.

- [ ] Скопировать IP.
- [ ] Скопировать RTSP.
- [ ] Скопировать пароль.
- [ ] Проверить таблицу `Все устройства`, поиск, фильтры и переход `На план`.
- [ ] Проверить, что заголовки колонок `Объект` и `Зона` в таблице `Все устройства` сортируют строки по названию объекта и зоны.
- [ ] Проверить, что активные верхние фильтры в `Все устройства` подсвечиваются красной рамкой.
- [ ] Проверить, что в таблице `Все устройства` рядом с устройством нет отдельной кнопки `Правка`, а `На план` остаётся единственным быстрым переходом на карту.
- [ ] Проверить, что после нажатия верхнего `+` в sidebar создаётся объект, а фокус переходит на план/canvas.
- [ ] Проверить, что при создании нового объекта автоматически создаётся первая зона `Новая зона`, она становится активной, а ветка объекта раскрывается.
- [ ] Проверить, что при создании помещения, устройства и текстового поля все стандартные подписи отображаются без крякозябр.
- [ ] Проверить все инструменты создания: `Камера`, `NVR`, `DVR`, `Switch`, `PoE Switch`, `Помещение`, `Текст`, `Кабель` - дефолтные имена, placeholder и подсказки должны быть нормальной русской строкой.
- [ ] Проверить, что значки на карте у камеры, switch/PoE/NVR/DVR, locked-элементов и locked-кабелей отображаются как нормальные SVG/ASCII-метки, без `рџ...` и других крякозябр.
- [ ] Проверить, что при активном инструменте `Помещение` курсор остаётся курсором рисования даже при наведении на другие помещения и объекты.
- [ ] Проверить, что новый `Room` можно начать рисовать поверх уже существующего помещения, и старое помещение не блокирует старт жеста.
- [ ] Проверить, что при рисовании стены линия свободно идёт в любом направлении, снапится к сетке и конец стены следует за курсором без прыжков.
- [ ] Проверить, что при перетаскивании одного конца стены стена вращается вокруг противоположного конца, а не только растягивается.
- [ ] Проверить, что у уже созданной стены перетаскивание конца даёт такой же живой resize/rotate, как у новой стены при рисовании.
- [ ] Проверить, что после нажатия `+` у строки объекта создаётся дочерняя зона/этаж, а фокус переходит на план/canvas.
- [ ] Проверить, что ручное переименование объекта и зоны через карандаш или double click остаётся редактируемым.
- [ ] Проверить, что в поле переименования объекта/зоны работают обычный ввод, Backspace, Delete и вставка текста без дополнительного клика по экрану.
- [ ] Проверить, что клик по плану после ввода имени объекта/зоны снимает фокус с поля, сохраняет новое имя и возвращает фокус на canvas.
- [ ] Проверить, что клик по правой панели или другой строке sidebar также завершает активное переименование, а название в правой панели меняется синхронно во время ввода.
- [ ] Проверить, что переименование объекта/зоны больше не пишет диагностические события в DevTools console.
- [ ] Проверить, что при выборе любого инструмента рисования текущее выделение на плане сразу сбрасывается и не остаётся подсвеченным.
